💾 백업/복구
플랫폼의 중요한 데이터를 정기적으로 백업하고, 필요 시 복구하는 페이지입니다. 전체 또는 선택적 백업으로 유연한 데이터 관리가 가능합니다. 자동 백업 설정으로 정기적인 백업을 자동화할 수 있으며, 백업 이력 관리로 특정 시점의 데이터 복구가 가능합니다. 월 1회 이상 정기 백업을 실시하여 데이터 손실을 방지하세요.

💾 데이터 백업/복구

📥 백업 생성
candidates, shared_files, blog_articles, admins 테이블을 JSON으로 저장합니다.
📤 백업 복구
이전에 생성한 백업 JSON 파일을 업로드해 데이터를 복원합니다 (upsert 방식).
⏰ 자동 백업 설정
⚠️ 자동 백업은 Supabase pg_cron 또는 서버 스케줄러 연동이 필요합니다. 현재는 설정값만 저장됩니다.
